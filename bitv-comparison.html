01
定价机制 Pricing Mechanism
标记价格 ⚠ 差异 指数价格 ⚠ 差异
▼
术语
● Gate.io
● Binance
● OKX
标记价格
Mark Price
⚠ 存在差异
Gate.io
中位数（价格1, 价格2, 最新成交价）
价格1 = 指数价格 × (1 + 资金费用基差率)
价格2 = 指数价格 + 移动平均基差（5分钟/每秒采样）
⚑ 防操纵：偏离均值过大时暂停更新
Binance
中位数（价格1, 价格2, 合约价格）
价格1 = 指数价格 × (1 + 资金费率×剩余时间/周期)
价格2 = 指数价格 + 30秒移动平均基差（每秒采样）
OKX
指数价格 + 基差移动平均值
基差 = (合约卖一价 + 买一价) / 2 − 指数价格
⚑ 无三值取中逻辑，结构更简洁
指数价格
Index Price
⚠ 存在差异
Gate.io
来源于多个交易所的加权平均现货价格
Binance
基于 16+ 交易所的加权平均
⚑ 单一来源偏差超 3% 时自动限幅
OKX
来源多个现货交易所，加权计算
02
盈亏计算 PnL Calculation
已实现盈亏 ⚠ 未实现盈亏 ✓ 仓位权益 ⚠
▼
术语
● Gate.io
● Binance
● OKX
仓位名义价值
Position Notional Value
✓ 基本一致
Gate.io
USDT永续合约
合约张数 × 合约乘数 × 标记价格
Binance
价格 × 数量
用于计算维持保证金分级
OKX
面值 × 张数 × 合约乘数 × 标记价格
仓位权益
Position Equity
⚠ 存在差异
Gate.io
未在单独术语中定义，体现在维持保证金率计算中
Binance
含义等同于账户余额 + 未实现盈亏
OKX
调整后权益（独立概念）
账户总权益（考虑未实现盈亏后）
⚑ 用于触发强平判断，定义最清晰
已实现盈亏
Realized PnL
⚠ 存在差异
Gate.io
当前仓位已产生的手续费、资金费用及减仓盈亏
平仓盈亏 + 已付手续费 + 已付资金费
Binance
取决于实际成交市场价格（无单独公式定义）
OKX
亏损仓位与减仓仓位抵消后实现的盈亏
未实现盈亏
Unrealized PnL
✓ 三家一致
Gate.io
多仓 = 张数 × 乘数 × (标记价格 − 开仓价格)
空仓 = 张数 × 乘数 × (开仓价格 − 标记价格)
不含手续费和资金费
Binance
基于标记价格计算，公式同 Gate 逻辑
OKX
USDT合约多仓 = 面值 × 张数 × 乘数 × (标记价格 − 开仓均价)
03
保证金体系 Margin System
维持保证金率 ⚠ 杠杆倍数 ⚠ 开仓保证金 ✓
▼
术语
● Gate.io
● Binance
● OKX
开仓保证金
Initial Margin
✓ 三家一致
Gate.io
杠杆交易开仓所需抵押的保证金金额
Binance
由所选杠杆倍数决定
初始保证金 = 名义价值 / 杠杆
OKX
初始保证金 = 名义价值 × 初始保证金率 (= 1/杠杆)
维持保证金率
Maintenance Margin Rate
⚠ 存在差异
Gate.io
维持保证金率 = 仓位保证金余额 / 仓位所需维持保证金
阶梯制：仓位越大，维持保证金率越高
Binance
维持保证金 = 名义价值 × 维持保证金率 − 速算数
阶梯制，含速算数简化计算
OKX
由仓位档位决定
⚑ 组合保证金模式下采用压力测试值
保证金（泛指）
Margin
⚠ 分类差异
Gate.io
保证金 = 起始保证金 + 平仓手续费
Binance
区分初始保证金（开仓）和维持保证金（持仓最低要求）
OKX
按账户模式分：逐仓保证金 / 全仓保证金 / 组合保证金
杠杆倍数
Leverage
⚠ 存在差异
Gate.io
可调整，调高后起始保证金降低但风险增加；更改杠杆不影响已实现盈亏
Binance
阶梯制：仓位名义价值越大，最大杠杆越低
⚑ 新账户前 30 天最高 20 倍限制
OKX
仓位档位越高，允许杠杆越低（档位递减逻辑）
04
资金费率 Funding Rate
费率公式 ✓ 结算周期 ⚠
▼
术语
● Gate.io
● Binance
● OKX
资金费率
Funding Rate
✓ 三家一致
Gate.io
= 溢价指数平均值 + clamp(间隔利息 − 溢价指数, ±0.05%)
预设日利率 0.03%，多空互付，平台不收取
Binance
同 Gate，使用深度加权价格计算溢价指数
OKX
同机制，多空双方互付，平台不收取
资金费结算周期
Settlement Period
⚠ 存在差异
Gate.io
默认 8H（UTC 0:00 / 8:00 / 16:00）
⚑ 费率触及上下限时自动缩短为 1H
Binance
通常 8H，部分合约 1H（固定，无动态调整）
OKX
通常 8H（无动态调整机制描述）
资金费
Funding Fee
✓ 三家一致
Gate.io
资金费 = 仓位价值 × 资金费率
逐仓从该仓位保证金扣除；全仓从账户权益扣除
Binance
同 Gate 逻辑，按标记价格计算仓位价值
OKX
同 Gate 逻辑，按标记价格计算仓位价值

仓位操作 Position Operations
开仓/平仓 ✓ 仓位模式 ⚠ 开仓均价 ⚠
▼
术语
● Gate.io
● Binance
● OKX
开仓
Open Position
✓ 三家一致
Gate.io
下单买入（做多）或卖出（做空），建立仓位
Binance
同，需满足初始保证金要求
OKX
同
平仓 / 平仓盈亏
Close Position
△ 细节差异
Gate.io
对持有仓位进行反向操作，结束持仓
平仓盈亏 = 含所有费用后的净盈亏（含手续费+资金费）
Binance
同（无单独盈亏公式定义）
OKX
同（无单独盈亏公式定义）
开仓均价
Average Entry Price
⚠ 命名差异
Gate.io
开仓价格（单笔成交价）
+ 开仓均价（多笔加权平均）两个概念并存
Binance
头寸价格（EP）= 开仓均价（统一为单一概念）
OKX
开仓均价，用于计算未实现盈亏和 ADL 排名
仓位模式
Position Mode
⚠ 存在差异
Gate.io
全仓：账户全部余额作为共同保证金，维持保证金率共用
逐仓：每仓独立保证金，亏损最多损失该仓保证金
Binance
全仓 WalletBalance / 逐仓 IsolatedWalletBalance
⚑ 逐仓时 TMM=0, UPNL=0
OKX
逐仓 / 全仓 / 组合保证金（三种模式）
⚑ 组合保证金采用压力测试，为三家最复杂
06
风控机制 Risk Control
强平价格 ⚠ 爆仓触发 ✓ ADL机制 ⚠
▼
术语
● Gate.io
● Binance
● OKX
强平价格
Liquidation Price
⚠ 公式差异
Gate.io
逐仓多头公式
(开仓均价 − 初始保证金/合约乘数/张数)
÷ (1 − 强平费率)
Binance
全仓含复杂矩阵公式（WB, TMM, UPNL 参数）
⚑ 需区分单向持仓/双向持仓模式，为三家最复杂
OKX
触发条件定义
账户权益 ≤ 维持保证金时触发
强制平仓（爆仓）
Liquidation
✓ 三家一致
Gate.io
保证金率 ≤ 100% 时触发，系统接管仓位，按破产价格分批减仓
Binance
同，触发后自动强平
OKX
维持保证金率达到或低于 100% 时触发
阶梯减仓
Tiered Liquidation
✓ 基本一致
Gate.io
强平触发后按破产价分批次平仓（非一次性清算）
⚑ 减仓过程中保证金率若恢复 >100% 则停止
Binance
类似逻辑，分批执行
OKX
分档位逐步强平
自动减仓
ADL · Auto-Deleveraging
⚠ 触发条件差异
Gate.io
保险基金耗尽时启动
排名 = 收益率 / 维持保证金率（从高到低强制撮合）
Binance
同机制，文档未独立详述触发条件
OKX
排名 = 杠杆收益率 = 收益率 / 账户保证金率
⚑ 风险保证金低于阈值时触发（非单纯"基金耗尽"）
